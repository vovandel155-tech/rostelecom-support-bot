# rostelecom-support-bot
Чат-бот для клиентской поддержки (Кейс-чемпионат Ростелекома)

Проект разработан в рамках кейс-чемпионата «ИТ-перспектива: решения для будущего с Ростелекомом» (Диплом III степени). 

[cite_start]Данное веб-приложение представляет собой комплексное решение для автоматизации службы технической поддержки[cite: 14]. [cite_start]Система включает в себя клиентский виджет на базе алгоритмов обработки естественного языка (NLP) [cite: 14][cite_start], панель оператора для обработки сложных запросов [cite: 16] [cite_start]и дашборд администратора для аналитики и управления базой знаний[cite: 16, 29].

## 🛠 Стек технологий

* [cite_start]**Язык:** Python 3.7+ [cite: 34]
* [cite_start]**Frontend / Пользовательский интерфейс:** Streamlit [cite: 19, 34]
* [cite_start]**Backend / Админ-панель:** Flask [cite: 25, 34]
* [cite_start]**База данных:** SQLite [cite: 32]
* [cite_start]**NLP & Аналитика:** NLTK, Pandas [cite: 34]

## 🌟 Ключевой функционал

Проект разделен на три функциональные зоны в зависимости от роли пользователя:

### 1. Пользовательский интерфейс (Streamlit)
* [cite_start]Общение с ботом через выбор категорий или свободный ввод текста[cite: 20].
* [cite_start]Обработка запросов с помощью выделения ключевых слов и определения намерений (intent)[cite: 14].
* [cite_start]Возможность эскалации диалога на живого оператора («Связаться с оператором»)[cite: 23].
* [cite_start]Система обратной связи: оценка ответа (👍/👎) и возможность предложить свой вариант для дообучения базы[cite: 24].

### 2. Панель оператора (Flask)
* [cite_start]Просмотр очереди эскалированных (сложных) запросов[cite: 26].
* [cite_start]Доступ к истории сообщений пользователя для контекста[cite: 15, 16].

### 3. Панель администратора
* [cite_start]**Аналитика:** Дашборды со статистикой запросов, топом тем и анализом неотвеченных сообщений[cite: 16]. [cite_start]Доступ защищен паролем[cite: 27].
* [cite_start]**Управление базой знаний (CRUD):** Добавление, редактирование и удаление пар «вопрос-ответ»[cite: 14, 29].
* [cite_start]Импорт и экспорт базы данных в форматах CSV и JSON[cite: 14, 29].

## ⚙️ Архитектура и потоки данных

[cite_start]Система построена на взаимодействии двух серверов (Streamlit и Flask)[cite: 32]. [cite_start]База знаний хранится в таблице `faq` (SQLite)[cite: 32]. Аналитика и обучение бота реализованы через систему логирования:
* [cite_start]`requests.log` — сбор всех входящих запросов[cite: 33].
* [cite_start]`escalations.log` — учет переводов на оператора[cite: 33].
* [cite_start]`feedback.log` / `suggestions.log` — сбор оценок и предложений пользователей для последующего анализа[cite: 33].
