# rostelecom-support-bot
Чат-бот для клиентской поддержки (Кейс-чемпионат Ростелекома)

Проект разработан в рамках кейс-чемпионата «ИТ-перспектива: решения для будущего с Ростелекомом» (Диплом III степени). 

Данное веб-приложение представляет собой комплексное решение для автоматизации службы технической поддержки. Система включает в себя клиентский виджет на базе алгоритмов обработки естественного языка (NLP), панель оператора для обработки сложных запросов и дашборд администратора для аналитики и управления базой знаний.

## Стек технологий

* **Язык:** Python 3.7+
* **Frontend / Пользовательский интерфейс:** Streamlit
* **Backend / Админ-панель:** Flask
* **База данных:** SQLite
* **NLP & Аналитика:** NLTK, Pandas

## Ключевой функционал

Проект разделен на три функциональные зоны в зависимости от роли пользователя:

### 1. Пользовательский интерфейс (Streamlit)
* Общение с ботом через выбор категорий или свободный ввод текста.
* Обработка запросов с помощью выделения ключевых слов и определения намерений (intent).
* Возможность эскалации диалога на живого оператора («Связаться с оператором»).
* Система обратной связи: оценка ответа (👍/👎) и возможность предложить свой вариант для дообучения базы.

### 2. Панель оператора (Flask)
* Просмотр очереди эскалированных (сложных) запросов.
* Доступ к истории сообщений пользователя для контекста.

### 3. Панель администратора
* **Аналитика:** Дашборды со статистикой запросов, топом тем и анализом неотвеченных сообщений. Доступ защищен паролем.
* **Управление базой знаний (CRUD):** Добавление, редактирование и удаление пар «вопрос-ответ».
* Импорт и экспорт базы данных в форматах CSV и JSON.

## ⚙️ Архитектура и потоки данных

Система построена на взаимодействии двух серверов (Streamlit и Flask). База знаний хранится в таблице `faq` (SQLite). Аналитика и обучение бота реализованы через систему логирования:
* `requests.log` — сбор всех входящих запросов.
* `escalations.log` — учет переводов на оператора.
* `feedback.log` / `suggestions.log` — сбор оценок и предложений пользователей для последующего анализа.
